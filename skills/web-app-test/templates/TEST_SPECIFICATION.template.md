- **设计策略**：单测 / 构建 / API 的程序测试脚本**必须预先设计并覆盖主要功能**；只有 E2E 的浏览器流程**不预先设计**，由 AI 测完判断需回归才沉淀为 📜。

## 测试分层（4 层，按种类命名）

| 层 | 含义 | 依赖后端运行 | 设计策略 |
|----|------|:---:|------|
| **单测** | 函数级逻辑，不联网不开浏览器（冒烟/环境健康、前后端纯函数、跨模块协作都归此） | 否 | 预先设计，覆盖主要功能 |
| **构建** | 前端生产构建 / 静态类型检查（`npm run build`、`tsc`、`eslint`） | 否 | 预先设计 |
| **API** | REST 接口契约与集成，依赖后端运行 | 是 | 预先设计，覆盖主要端点 |
| **E2E** | Playwright 驱动真实浏览器、连真实后端，模拟用户操作 | 是 | 不预先设计，AI 动态接入、测完判断需回归再沉淀 |

> 单测 + 构建 + API = 预先设计的程序测试脚本（回归底座）；E2E = 动态接入、按需沉淀。
