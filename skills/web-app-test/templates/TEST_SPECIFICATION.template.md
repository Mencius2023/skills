## L4 浏览器 E2E 测试（真实后端）

> 目标：模拟用户完整操作流程，连接真实后端，不做 mock。前后端集成行为与 API 契约同时验证。
> 依赖：后端运行在 `localhost:<后端端口>`（`<后端启动命令>`）；浏览器用例还需 Chromium（`npx playwright install chromium`）。
> 注意：涉及耗时外部调用（如 LLM）的用例超时设 180s+；对不确定输出的断言用「至少 N 个」而非精确数。
>
> **L4 内部再分两层，设计策略不同（与 SKILL.md「测试分类」一致）：**
>
> | 子类 | 是否经浏览器 | 设计策略 | 标记 | 可执行资源 |
> |------|------------|---------|------|-----------|
> | **API 契约**（纯 HTTP） | 否，纯 HTTP 请求 | **预先设计**：属程序测试，落地即写进 spec，覆盖主要端点 | 📜 | `scripts/<flow>.spec.js` 的 API 契约部分 |
> | **浏览器流程**（模拟用户） | 是，Playwright 驱动浏览器 | **不预先设计**：默认 AI 动态接入现场操作；测完判断需回归才沉淀为 spec | 🤖→📜 | 现场动态 / 沉淀后进 spec |
>
> 即：📜 标记里，**API 契约用例落地即写**；**浏览器流程用例**初始是 🤖（动态），仅在 AI 测完判断「后续需回归且流程稳定」后才升级为 📜 并补进 spec。📜 部分可由 `run_regression.py` 或 `npx playwright test --config scripts/playwright.config.js` 批量复跑；🤖 部分由 AI 读 `L4_OPERATION_GUIDE.md` 现场执行。
